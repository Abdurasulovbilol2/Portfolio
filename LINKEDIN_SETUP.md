### Method 1: GitHub Pages (Recommended - Free)

#### Step 1: Create GitHub Repository
1. Go to [GitHub.com](https://github.com) and sign in to your account
2. Click **"New Repository"** or the **"+"** icon
3. Name your repository: `qa-automation-portfolio`
4. Make it **Public** (required for free GitHub Pages)
5. Click **"Create repository"**

#### Step 2: Upload Your Portfolio Files
1. Click **"uploading an existing file"** link
2. Drag and drop all files from your `QA` folder:
   - `index.html`
   - `styles.css`
   - `script.js`
   - `test-artifacts/` folder (with all HTML files inside)
   - `README.md`
3. Write commit message: "Add QA automation portfolio"
4. Click **"Commit changes"**

#### Step 3: Enable GitHub Pages
1. Go to your repository **Settings** tab
2. Scroll down to **"Pages"** section (left sidebar)
3. Under **"Source"**, select **"Deploy from a branch"**
4. Choose **"main"** branch and **"/ (root)"** folder
5. Click **"Save"**
6. Wait 2-3 minutes for deployment

#### Step 4: Get Your Live URL
Your portfolio will be available at:
```
https://YOUR_USERNAME.github.io/qa-automation-portfolio
```
Replace `YOUR_USERNAME` with your actual GitHub username.
